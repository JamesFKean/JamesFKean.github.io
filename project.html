How Has China's Energy Consumption Changed As Its Economy Has Grown?

Aims

Since the 1980s China’s economy has been on a path of rapid and consistent growth. This process has turned China from a small and relatively impoverished economy, to one of the worlds’ largest, all of which needed to be powered by a prodigious rise in national energy consumption. That had profound consequences not just for the global economy, but for future environmental sustainability.

This project will demonstrate the paths countries rates of change in energy consumption and GDP have taken overtime, to show how China’s growth story is unique in the modern world. This will give perspective on how consistent growth leads to large changes overtime. In terms of total energy consumption, output, and emissions. Showing also how not all countries have had the same success in the transmutation of energy consumption into GDP.

Data Used

The data used in this project, mainly comes from BP and the World Bank. Data from the World Bank has been accessed using their provided Python library that has an inbuilt API, allowing for the easy calling of data series for desired countries over desired intervals of time. It does however require a large amount of formatting to get into a tidy format, suitable for visualisations.

The data from BP has been called from their website using requests. The data is first downloaded in excel format, then each of its separate sheets are passed to a Pandas Dataframe, with them being sorted by shape such time series data and stock data can be treated differently given the shape of the frame. The format of this allows for sheets to be called using a key value paring, which then allows for specific countries or data series to be called from that dictionary.

The China geoshape trellis scrapes data from three sources, SP Global and a Wikipedia table in html format. Then a PDF file using Camelot.

All data calling is automated, meaning that running the associated workbooks will get generate files that allow for recreation of the imbedded vega graphs.

Challenges in Data cleaning

The largest hurdle to processing and cleaning the data was ensuring that the calling, sorting and structuring of BP energy data without having to resort to exporting to external files and undertaking processing outside of the collab environment. This involved filtering sheets by shape, as sheets containing time series data skip a different number of rows to those containing stock data. Filtering these sheets into separate elements is useful as it means that time series sheets can be called from the dictionary of data frames. Then frames can be manipulated to get data for all the countries or just one.

Scarping the data used in the China trellis chart required some heavy cleaning due to the way in which it was nested within the HTML. However, using some of the built in Python commands allowed for the data to be formatted in a way that would allow for it to be passed to a Pandas object while retaining its table like structure. Manipulating the columns back in also required some cleaning, along with ensuring that all the provincial names matched between the three sources.

Scraping from the PDF required trialing of several libraries, with Camelot eventually producing the best results.
